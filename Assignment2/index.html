Servers
  • Standard x86 64-bit servers or Power Linux nodes that run supported Linux distributions.
  • A high CPU count promotes better cloud tiering throughput because although object storage can be slow in I/O operations per thread, object storage can support many threads. Use sixteen or more CPUs when you select your hardware.
  • Cloud tiering services demand a large amount of memory, which is why the minimum recommended memory size is 128 GB. Memory size requirements increase if the number of files increases, as you add files on the cloud means you must increase the memory that is on your system. It is recommended that you use 10 - 20 times as much memory that is required so the Cloud services can cache its directory database data.
